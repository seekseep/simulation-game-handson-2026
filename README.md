これは学習用のプロジェクトです。
docs のなかに各ステップごとにREADMEがあるからそれを作るようにしたい。

大きく分けて４つの章にわかれる
docs/1-practice
docs/2-ui
docs/3-api
docs/4-ai

docs/1-practice　だけは
practice というディレクトリを作っていくつかのファイルを書きながら学んでいくスタイルにしたい。

いかの３つは一つのプロジェクトとして作っていく
docs/2-ui
docs/3-api
docs/4-ai

ハンズオンなので以下の内容について追記する
- 作るファイル
- 削除するファイル
- 変更する内容
- 変更するファイル名

各ステップ

docs/{章番号}-{章の名称}/{節番号}-{節の名称}/README.md
となっていて
docs/{章番号}-{章の名称}/{節番号}-{節の名称}/examples/{サンプル番号}
がある

docs/{章番号}-{章の名称}/{節番号}-{節の名称}/README.md　のなかは
実装のものに関してはサンプルごとに 1レベルの見出しをつける



次のように各サンプルごとにかく

# タイトル
実装例: `/examples/{サンプル番号}`

## 目標

## 作業前のディレクトリ構造

```
{tree コマンドの出力}
```

## 作業

1. {作業名}
2. {作業名}

## {作業名}

ファイル名: `/path/to/file`

## 動作確認

{期待する結果}

- {動作確認内容1}
- {動作確認内容2}

## 作業後のディレクトリ構造

```
{tree コマンドの出力}
```
